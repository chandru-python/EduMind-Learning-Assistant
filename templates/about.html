Dynamic Vector Indexing Layer

The engine loads document knowledge chunks out of specialized serialized chunks.pkl and embeddings.pkl objects directly into memory maps upon backend bootup.

Dependency: pickle, scikit-learn, numpy
👁️‍🗨️

BGE-Small Dense Semantic Parser

Rather than matching text using simple keywords, our retrieval system evaluates text embeddings using the state-of-the-art open-source BAAI/bge-small-en-v1.5 transformer model. This allows the system to understand deep conceptual relationships between physics formulas and complex biological systems.

Dependency: sentence-transformers
🛡️

Deterministic Prompt Anchoring Guardrails

To minimize the risk of AI hallucination, our prompt engineering restricts the TinyLlama-1.1B-Chat-v1.0 model. If the top-ranked vector search results score below strict cosine similarity thresholds, the system halts generation and outputs a clean, safe fallback warning.

Logic Constraint: do_sample=False, truncation=True

Frequently Asked Engineering Questions

Deep operational data parameters and architecture answers.

How are the educational text source datasets structured?

STEM textbooks and study notes are broken down into overlapping paragraph segments. These segments are processed through an embedding pipeline using the Sentence-Transformers library and saved as vectorized index structures within portable Python pickle binary archives.

Why use local hardware inference instead of vendor APIs?

Running our stack on local compute threads allows us to eliminate third-party API dependencies, protect user data privacy, and maintain full control over our model runtime environments without incurring per-token commercial costs.

How does the system prevent model hallucinations?

The `generate_answer` engine evaluates the top context chunks returned by the retriever. The system-level prompt includes strict parameters that forbid the model from answering unless the solution is explicitly present within the provided context.
